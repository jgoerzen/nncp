Release 5.3.0

  • Progress messages contain prefix, describing the running action.
  • Fixed not occurring handshake messages padding.
  • Finish all SP protocol related goroutines, less memory leak.
  • SP protocol generates less socket write calls, thus generating less TCP packets.
  • Check onlinedeadline and maxonlinetime options every second, independently from socket reads (up to 10 seconds).
  • Once per minute, if no other traffic exists, PING packets are sent in SP-connection. That allows faster determining of connection unworkability.
  • nncp-toss uses lock-file to prevent simultaneous tossing.
